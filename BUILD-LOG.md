# Build Log

This file tracks issues, decisions, and improvement suggestions across builds.
The `/improve` command reviews this log to suggest system updates.

---

## Template for Each Build

Copy this template when starting a new build:

```markdown
---

## Build: [DATE] - [Project Name]

### Phase: [Discovery | Spec | Plan | Code | QA | RLS]

### Issues Encountered
- [ ] [Issue description] - [Resolution/status]

### Questions Asked Mid-Build
- [Question that should have been in intake]
- [Answer received]

### Manual Overrides
- [What was changed from default/template and why]

### Patterns Reused
- [Pattern name] from [source]

### New Patterns Discovered
- [Description] - Consider adding to skills

### Context Management
- Peak context usage: [X]%
- Compaction needed: [Yes/No]
- Sessions used: [Number]

### RLS Notes (if applicable)
- Tables secured: [X] of [Y]
- Policies that needed adjustment: [List]

### Suggested System Updates
- [ ] [Specific suggestion for /improve to consider]

---
```
